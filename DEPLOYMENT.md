## Deployment to Hostpoint (Apache Shared Hosting)

### What to upload
Upload the **entire contents** of the `dist/` folder (not the folder itself) to the domain's web root:
- For the main domain: upload to `public_html/` or `www/`
- For a subdomain: upload to the subdomain's document root

### Required files
The `.htaccess` file is included in `public/` and will be copied to `dist/` automatically during build. It is **essential** — without it, all routes except `/` will return a 404 on page refresh.

### Upload method
Use FTP/SFTP (FileZilla or similar) or Hostpoint's File Manager.

### Upload checklist
- [ ] `index.html`
- [ ] `assets/` folder (JS + CSS bundles)
- [ ] `images/` folder
- [ ] `videos/` folder
- [ ] `.htaccess` ← **critical for React Router**

## React Router Configuration

The project uses client-side routing (React Router v7). On Apache shared hosting, all requests must be rewritten to `index.html`.

The file `public/.htaccess` (included in the repo) handles this automatically:

```apache
Options -MultiViews
RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^ index.html [QSA,L]
```

This file is copied to `dist/.htaccess` automatically by `vite build`.
If it is missing after upload, routes like `/work/esnr` will return 404 on direct access or refresh.

## Custom Domain Setup (digital-twilight.com)

1. Upload site files to Hostpoint's web root for the domain
2. In Hostpoint control panel, ensure the domain points to the folder containing `index.html`
3. Enable HTTPS (Hostpoint provides free Let's Encrypt SSL — activate in the control panel)
4. Test all routes by navigating directly to e.g. `https://digital-twilight.com/work/esnr`
